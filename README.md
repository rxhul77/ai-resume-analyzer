# AI Resume Analyzer

AI Resume Analyzer is a Flask-based web application that compares a PDF resume with a job description using Google Gemini AI.

It generates an ATS-style report containing the resume match percentage, missing skills, strengths, weaknesses, and suggestions for improvement.

## Live Demo

[Open AI Resume Analyzer](https://ai-resume-analyzer-rq53.onrender.com/)

## Features

* Upload a resume in PDF format
* Paste a job description
* Extract text from the uploaded PDF
* Generate an AI-powered ATS analysis
* Display ATS match percentage
* Identify missing skills
* Show resume strengths and weaknesses
* Provide improvement suggestions
* Responsive user interface
* Loading spinner during analysis
* Reset button
* Error handling for invalid files and API errors

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/rxhul77/ai-resume-analyzer.git
```

### 2. Open the project directory

```bash
cd ai-resume-analyzer
```

### 3. Create a virtual environment

```bash
python -m venv venv
```

### 4. Activate the virtual environment

For Windows PowerShell:

```powershell
.\venv\Scripts\Activate.ps1
```

For Windows Command Prompt:

```cmd
venv\Scripts\activate
```

For macOS or Linux:

```bash
source venv/bin/activate
```

### 5. Install the required packages

```bash
pip install -r requirements.txt
```

### 6. Create a `.env` file

Create a file named `.env` in the main project folder:

```env
GEMINI_API_KEY=your_gemini_api_key
```

Replace `your_gemini_api_key` with your actual Google Gemini API key.

## Run Locally

Start the Flask application:

```bash
python app.py
```

Open the following address in your browser:

```text
http://127.0.0.1:5000
```

## How It Works

1. The user uploads a resume in PDF format.
2. PyPDF extracts text from the resume.
3. The user enters a job description.
4. The extracted resume text and job description are sent to Gemini AI.
5. Gemini generates an ATS-style analysis.
6. Python Markdown converts the response into formatted HTML.
7. Flask displays the final report on the webpage.

## Deployment on Render

The application is deployed using Render.

### Render configuration

**Build Command**

```bash
pip install -r requirements.txt
```

**Start Command**

```bash
gunicorn app:app
```

The following environment variable must be added in Render:

```text
GEMINI_API_KEY
```

Its value should be your actual Gemini API key.

## Security

* The Gemini API key is stored in an environment variable.
* The `.env` file is excluded from GitHub through `.gitignore`.
* Uploaded filenames are sanitized using `secure_filename`.
* Only PDF files are accepted by the upload form.

## Current Limitation

The application currently checks whether the uploaded file has a `.pdf` extension, but it does not fully verify whether the PDF is actually a resume.

A non-resume PDF may still be processed and produce an inaccurate analysis.
